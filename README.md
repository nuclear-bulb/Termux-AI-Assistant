#Termux-AI 助理

**18岁大学生在手机Termux上构建的私人AI，带记忆库、熔断机制和记忆强隔离。**

## 背景

在安卓 Termux 终端环境中，调用 DeepSeek API 部署了一个带记忆机制和故障处理能力的个人 AI 对话系统。以下是四个版本的迭代过程。

## 第一代：基础对话

1. 在 Termux 中配置 Python 环境
2. 接入 DeepSeek API，实现命令行对话
3. 验证 API 调用的可行性与稳定性

## 第二代：黑匣子记忆

增加跨对话记忆功能：

1. **记忆载体**：本地 JSON 文件
2. **加载**：启动时自动读取历史记录，注入对话
3. **保存**：退出时询问是否保存（y/n，默认不保存）
4. **清空**：支持一键清除全部记忆
5. **撤销**：支持撤销上一轮对话

## 第三代：故障日志

增加异常处理与记录功能：

1. **故障日志**：API 错误或网络异常时，自动记录时间戳和错误详情
2. **异常恢复**：请求失败时移除未完成的用户消息，保持对话状态一致

## 第四代：强隔离与智能熔断

增加运行模式控制与自动熔断：

1. **强隔离模式**：`isolate` 命令开启，AI 暂不访问历史记忆；`restore` 恢复
2. **智能熔断**：检测到连续异常请求时自动熔断，防止 Token 雪崩
3. **状态查询**：`status` 命令查看当前运行状态

## 当前系统功能一览

- 记忆管理：加载、保存、清空、撤销
- 故障处理：错误日志、异常恢复
- 模式切换：强隔离与正常模式
- 熔断保护：自动止损

## 经验总结

Termux 可作为轻量级 AI 应用开发环境。本地记忆文件 + API 调用的组合方式，实现了跨对话记忆的下一次启动即可用，无需依赖服务端持久化。熔断与隔离机制提升了系统稳定性。整个过程验证了个人在移动端构建定制化 AI 对话系统的可行性。
